# Google グループ一括管理ツール

Google スプレッドシートを管理画面として、Google Workspace の Google グループ(1,000 超)の作成・削除、メンバー管理、設定変更、棚卸しを一括実行する GAS ツール。

対応ドキュメント:

- `グループ一括管理GAS_要件定義書_v0.1.md`
- `グループ一括管理GAS_設計書1_シート設計_v1.0.md`
- `グループ一括管理GAS_設計書2_処理方式設計_v0.1.md`

## セットアップ手順

実行者は Google Workspace 特権管理者であること(P-1)。

### 1. スプレッドシートと GAS プロジェクトの作成

1. 管理用スプレッドシートを新規作成し、コンテナバインドの Apps Script プロジェクトを作成する。
2. `.clasp.json` に scriptId を設定し、`clasp push -f` でデプロイする(`appsscript.json` が拡張サービス・スコープを宣言済み。manifest を更新する push は確認プロンプトで止まるため `-f` を付ける)。

### 2. 標準 GCP プロジェクトへの切り替え(必須)

本ツールは Cloud Identity API を `UrlFetchApp` による REST で呼び出すため、スクリプトをユーザー管理の GCP プロジェクト(標準プロジェクト)に紐づける必要がある。**デフォルトプロジェクトのままでは Cloud Identity API を有効化できず、種別判別が `HTTP 403`(`Cloud Identity API has not been used in project ...`)で失敗する。**

1. GCP コンソールで同一組織のプロジェクトを用意し、「プロジェクト番号」(数字)を控える。
2. そのプロジェクトで以下の **4 つの API をすべて有効化**する(標準プロジェクトでは拡張サービス・組み込みサービスの API も自動有効化されない):
   - **Admin SDK API**(グループ・メンバー操作)
   - **Groups Settings API**(設定 27 項目の取得・変更)
   - **Cloud Identity API**(動的・セキュリティグループの判別)
   - **Google Drive API**(CSV のドライブ出力。組み込みサービス `DriveApp` も標準プロジェクトでは有効化が必要)
3. 同プロジェクトの「OAuth 同意画面」を構成する(ユーザータイプ「内部」、アプリ名等の入力のみで可)。
4. Apps Script エディタ → 歯車「プロジェクトの設定」→「Google Cloud Platform(GCP)プロジェクト」→「プロジェクトを変更」→ 1 の番号を入力する。
5. 次回実行時に再認可(OAuth 同意)を求められるので承認する。

### 3. 管理コンソール側の確認

管理コンソール > セキュリティ > API の制御 で、Admin SDK 等への API アクセスがブロックされていないことを確認する。

### 4. 初期セットアップと設定

1. スプレッドシートを再読み込みし、メニュー「グループ一括管理 > メンテナンス > 初期セットアップ」を実行する(9 シート・入力規則・マスタ選択肢を自動生成)。
2. 設定シートに `CSV_FOLDER_ID`(CSV 出力先フォルダ ID)などを入力する。
3. 動作確認は読み取り系から行う: 「棚卸し > グループ一覧を取得」→「設定一覧を取得」の順に実行し、ログシートで完了を確認する。

## 使い方

- **指示の実行:** 指示_グループ / 指示_メンバー / 指示_設定変更 に操作行を記入し、「ドライラン」で検証してから「本実行」を実行する。結果・メッセージ・実行日時・実行 ID が各行に書き戻される。
- **再実行のルール:** 結果が「空欄 / 検証OK / 検証NG / 失敗」の行のみ処理対象。「成功 / スキップ / 対象外」は再実行されない(冪等)。再処理したい行は「結果クリア(選択範囲)」を使う。
- **棚卸し:** 「棚卸し > グループ一覧を取得 / メンバー一覧を取得 / 設定一覧を取得」。シートは最新スナップショットに上書きされ、完了時に CSV が自動出力される(D-2)。メンバー一覧・設定一覧は**棚卸_グループのスナップショットを入力**とするため、先に「グループ一覧を取得」を実行する(スナップショット取得後に削除されたグループは「スキップ」として集計される)。
- **予約・定期:** 設定シートの `RESERVE_DATETIME` / `RESERVE_TARGET` / `INVENTORY_SCHEDULE` を入力して「予約・定期」メニューから登録する。予約実行は本実行のみ(ドライラン不可)。
- **分割実行:** `TIME_LIMIT_SEC`(既定 300 秒)を超えると自動で中断し、約 1 分後に継続トリガーで再開する。進捗はログシートで確認できる。

## 実装メモ(設計からの確定事項)

設計書 2/2 の残課題 2 点を実装時に以下のとおり確定した。

1. **Cloud Identity の呼び出し方式:** Cloud Identity Groups API には GAS 拡張サービスが存在しないため、`UrlFetchApp` + `ScriptApp.getOAuthToken()` による REST 呼び出しで実装した。これに伴い manifest に `script.external_request` スコープを追加している。
2. **顧客 ID の取得:** Cloud Identity `groups.list` の `parent` に必要な顧客 ID(`customers/Cxxxx`)を `Directory API Customers.get('my_customer')` で取得するため、`admin.directory.customer.readonly` スコープ(読み取り専用)を追加している。
3. **マスタ_選択肢の初期値:** Groups Settings API リファレンスの列挙値を `01_Config.gs`(`SETTINGS_FIELDS`)に転記して確定。初期セットアップがマスタシートへ自動投入する。

その他の補足:

- 入力規則は「警告表示」(`setAllowInvalid(true)`)とし、CSV 貼り付けを妨げない。値の検証はスクリプト側で必ず行う(FR-09 の二重チェック)。
- ドライランでも動的・セキュリティグループは「対象外」を書き込む(設計書 2/2 6 章。対象外の行は本実行・ドライランとも処理対象外のため整合する)。
- 異常終了したジョブが残った場合は「メンテナンス > 実行中ジョブの中断」を 2 回実行すると状態を強制クリアできる(稼働中のジョブは最終更新から 10 分間は保護される)。

## トラブルシューティング

| 事象 | 原因と対処 |
|------|-----------|
| `clasp push` で `Service not found: groupssettings ...` | manifest の Groups Settings のバージョン誤り。`"serviceId": "groupssettings"` の `version` は `"v1"`(`directory_v1` のような形式ではない)。 |
| `CloudIdentity: HTTP 403: Cloud Identity API has not been used in project ...` | スクリプトがデフォルト GCP プロジェクトに紐づいたまま。エラー中のプロジェクト番号は Apps Script の内部プロジェクトで、別プロジェクトで API を有効化しても解消しない。セットアップ手順 2 の「標準 GCP プロジェクトへの切り替え」を実施する。 |
| 設定一覧で `GroupsSettings.get: A system error has occurred` | Groups Settings API は存在しないグループに 404 ではなくこの汎用エラーを返す。棚卸_グループのスナップショット取得後に削除されたグループが原因(ツールは Directory で存在確認し「スキップ」として集計する)。「グループ一覧を取得」で最新化すれば解消。 |
| 実行時に「実行中(または異常終了した)ジョブが存在します」 | エラー停止したジョブの `JOB_STATE` が残っている。「メンテナンス > 実行中ジョブの中断」を実行(異常終了したジョブは 2 回実行で強制クリア。稼働中ジョブは最終更新から 10 分間保護)。 |
| ジョブは完了するが失敗件数が計上される | ログシートの備考に「失敗あり(直近: グループ名: エラー内容)」が記録されるので、その内容から原因を特定する。 |

## 開発

```bash
npm run lint     # ESLint(リポジトリルートで実行)
npm run format   # Prettier
clasp push -f    # 本ディレクトリで実行(manifest 更新時は確認プロンプトで止まるため -f を付ける)
```
