## 1. 本書の位置づけ

設計フェーズの第 1 分冊として、管理用スプレッドシートのシート構成と列定義を定める。処理方式(バッチ・継続実行、予約・定期トリガー、API 対応、エラー処理)は第 2 分冊「処理方式設計」で定める。

## 2. スプレッドシート構成

1 つのスプレッドシートに以下のシートを持つ。CSV はシートとしては持たず、棚卸し・実行結果をドライブへファイル出力する(FR-08)。シート名・列見出しは日本語とし、内部キーは英語で実装する(D-3)。

| # | シート名 | 用途 | 主な対応要件 |
|---|----------|------|--------------|
| 1 | 設定 | 動作パラメータ(バッチサイズ、予約日時、CSV 出力先など) | NFR-08, FR-14, FR-15 |
| 2 | 指示_グループ | グループの作成・削除の指示 | FR-02, FR-03 |
| 3 | 指示_メンバー | メンバーの追加・削除の指示 | FR-04, FR-05 |
| 4 | 指示_設定変更 | グループ設定(27 項目)の変更指示 | FR-06 |
| 5 | 棚卸_グループ | 全グループ一覧の出力先 | FR-07 |
| 6 | 棚卸_メンバー | 全メンバー一覧の出力先 | FR-07 |
| 7 | 棚卸_設定 | グループ設定 27 項目の一覧出力先(別メニューによる任意実行。D-1) | FR-07 拡張・FR-06 支援 |
| 8 | ログ | 実行履歴 | FR-12, NFR-06 |
| 9 | マスタ_選択肢 | プルダウン(入力規則)の選択肢定義 | FR-09 |

シート・見出し・入力規則・書式は、カスタムメニューの「初期セットアップ」機能でスクリプトが自動生成する(手作業構築によるズレを防止。NFR-08 対応)。

## 3. 共通仕様

### 3.1 指示シートの基本ルール

- 1 行目は見出し(固定)、2 行目以降がデータ。1 行 = 1 操作とし、上の行から順に処理する。
- 結果列(結果・メッセージ・実行日時・実行 ID)はスクリプトが書き込む。手動編集は行わず、再実行したい場合はメニューの「結果クリア」を使用する。
- 設定変更シートは「記入されたセルのみ適用、空欄は現状維持」とする(項目単位の差分適用)。

### 3.2 結果ステータスの定義と処理対象ルール

| 結果値 | 意味 | 本実行の対象 | ドライランの対象 |
|--------|------|:---:|:---:|
| (空欄) | 未処理 | ○ | ○ |
| 検証OK | ドライランで問題なし | ○ | ○ |
| 検証NG | 事前バリデーションでエラー(修正後そのまま再実行可) | ○ | ○ |
| 失敗 | API 実行時エラー(再実行で再試行) | ○ | ○ |
| 成功 | 適用済み | − | − |
| スキップ | 冪等スキップ(追加済み・削除済みなど) | − | − |
| 対象外 | 動的・セキュリティグループのため非適用(FR-09) | − | − |

- ドライランは結果列に「検証OK / 検証NG」を書き込み、メッセージに「[ドライラン]」を付す。変更系 API は呼び出さない。
- 冪等性(NFR-04):「成功・スキップ・対象外」の行は再実行されない。

### 3.3 共通の結果列

すべての指示シートの末尾に以下の 4 列を置く。

| 列名 | 内容 |
|------|------|
| 結果 | 3.2 の結果値 |
| メッセージ | エラー内容・スキップ理由など |
| 実行日時 | 処理した日時 |
| 実行ID | ログシートと対応する ID(例: J20260612-093000) |

### 3.4 入力規則(プルダウン)

操作種別・ロール・設定の列挙値・削除確認は、マスタ_選択肢シートを参照する入力規則(プルダウン)を設定し、入力ミスを防止する。スクリプト側でも同じ値域を検証する(FR-09 の二重チェック)。

### 3.5 棚卸しシートの保存方式(D-2)

棚卸_グループ・棚卸_メンバー・棚卸_設定は、実行のたびにシートをクリアして最新スナップショットへ上書きする。履歴は実行完了時に自動で CSV 出力して保管する(セル数上限対策。詳細は第 2 分冊の CSV 出力仕様)。

## 4. 各シートの列定義

### 4.1 設定

| 列 | 項目 |
|----|------|
| A | キー |
| B | 値 |
| C | 説明 |

設定キー(初期値は第 2 分冊で最終化):

| キー | 値の例 | 説明 |
|------|--------|------|
| TIME_LIMIT_SEC | 300 | 1 実行の打ち切り秒数(6 分上限への安全マージン) |
| BATCH_ROWS | 50 | 結果を書き戻すチャンク単位の行数 |
| RETRY_MAX | 5 | レート制限時のリトライ回数 |
| RETRY_BASE_MS | 1000 | 指数バックオフの初期待機ミリ秒 |
| CSV_FOLDER_ID | (フォルダ ID) | CSV 出力先の Google ドライブフォルダ |
| RESERVE_DATETIME | 2026/06/20 09:00 | 予約実行の日時(FR-14) |
| RESERVE_TARGET | すべて | 予約実行の対象(すべて / グループ / メンバー / 設定変更) |
| INVENTORY_SCHEDULE | 毎週 月 07:00 | 定期棚卸しの頻度(FR-15。トリガー登録時に参照) |
| INVENTORY_MEMBERS | TRUE | 定期棚卸しでメンバー一覧も取得するか |
| LOG_RETENTION_DAYS | 365 | ログ保持日数(NFR-06) |

### 4.2 指示_グループ(作成・削除)

| 列 | 項目 | 必須 | 入力形式・備考 |
|----|------|------|----------------|
| A | 操作 | ○ | プルダウン:作成 / 削除 |
| B | グループメール | ○ | メールアドレス |
| C | グループ名 | 作成時○ | 文字列 |
| D | 説明 | − | 文字列 |
| E | 削除確認 | 削除時○ | プルダウン:「削除する」を選択した行のみ削除を実行(FR-03) |
| F〜I | 結果 / メッセージ / 実行日時 / 実行ID | − | スクリプトが書込 |

### 4.3 指示_メンバー(追加・削除)

| 列 | 項目 | 必須 | 入力形式・備考 |
|----|------|------|----------------|
| A | 操作 | ○ | プルダウン:追加 / 削除 |
| B | グループメール | ○ | メールアドレス |
| C | メンバーメール | ○ | ユーザーまたはグループのメールアドレス |
| D | ロール | − | プルダウン:MEMBER / MANAGER / OWNER(追加時のみ有効。空欄は MEMBER) |
| E〜H | 結果 / メッセージ / 実行日時 / 実行ID | − | スクリプトが書込 |

### 4.4 指示_設定変更(27 項目)

A 列で対象グループを指定し、B〜AB 列のうち記入されたセルのみを適用する。各列挙値の選択肢一覧は第 2 分冊の API 対応表に基づきマスタ_選択肢へ保持する。

| 列 | 項目 | 入力形式 |
|----|------|----------|
| A | グループメール(対象) | メールアドレス(必須) |
| B | email(変更後アドレス) | メールアドレス |
| C | name | 文字列 |
| D | description | 文字列 |
| E | whoCanJoin | 選択 |
| F | whoCanViewMembership | 選択 |
| G | whoCanViewGroup | 選択 |
| H | allowExternalMembers | TRUE / FALSE |
| I | whoCanPostMessage | 選択 |
| J | allowWebPosting | TRUE / FALSE |
| K | messageModerationLevel | 選択 |
| L | spamModerationLevel | 選択 |
| M | replyTo | 選択 |
| N | customReplyTo | メールアドレス |
| O | includeCustomFooter | TRUE / FALSE |
| P | customFooterText | 文字列 |
| Q | sendMessageDenyNotification | TRUE / FALSE |
| R | defaultMessageDenyNotificationText | 文字列 |
| S | membersCanPostAsTheGroup | TRUE / FALSE |
| T | includeInGlobalAddressList | TRUE / FALSE |
| U | whoCanLeaveGroup | 選択 |
| V | whoCanContactOwner | 選択 |
| W | whoCanModerateMembers | 選択 |
| X | whoCanModerateContent | 選択 |
| Y | whoCanAssistContent | 選択 |
| Z | enableCollaborativeInbox | TRUE / FALSE |
| AA | whoCanDiscoverGroup | 選択 |
| AB | defaultSender | 選択(DEFAULT_SELF / GROUP) |
| AC〜AF | 結果 / メッセージ / 実行日時 / 実行ID | スクリプトが書込 |

※ email(B 列)の変更はグループのリネームに相当する。記入されたセルのみ適用する方式のため追加の確認列は設けないが、運用上の注意としてマニュアルに明記する。

### 4.7 棚卸_設定(任意実行。D-1)

メニュー「棚卸し > 設定一覧を取得」で実行する。B〜AB 列は指示_設定変更と同一の並びとし、現在値を出力する。

| 列 | 項目 | 備考 |
|----|------|------|
| A | グループメール | |
| B〜AB | 設定 27 項目(現在値) | 指示_設定変更の B〜AB と同一の並び |
| AC | 取得日時 | |

※ 指示_設定変更と列並びを揃えているため、「棚卸_設定の行をコピーして指示_設定変更へ貼り付け、変更したい項目だけ残して(他を空欄にして)適用する」運用が可能。

## 5. カスタムメニュー構成

```
グループ一括管理
├ ドライラン
│ ├ グループ作成・削除
│ ├ メンバー追加・削除
│ └ 設定変更
├ 本実行
│ ├ グループ作成・削除
│ ├ メンバー追加・削除
│ ├ 設定変更
│ └ すべての指示を実行
├ 棚卸し
│ ├ グループ一覧を取得
│ ├ メンバー一覧を取得
│ ├ 設定一覧を取得(任意・D-1)
│ └ CSVへ出力
├ 予約・定期
│ ├ 予約実行を登録 / 解除(設定シートの日時・対象を参照)
│ ├ 定期棚卸しを登録 / 解除
│ └ 登録状況を確認
└ メンテナンス
  ├ 実行中ジョブの状態確認 / 中断
  ├ 結果クリア(選択範囲)
  └ 初期セットアップ(シート自動生成)
```
