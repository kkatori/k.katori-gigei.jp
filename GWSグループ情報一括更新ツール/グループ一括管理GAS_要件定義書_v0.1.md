# Google グループ一括管理ツール 要件定義書

**版数:** v0.1(ドラフト) **作成日:** 2026-06-12 **ステータス:** レビュー中

## 1. 背景・目的

Google Workspace 上の Google グループ(1,000 超)に関する作成・削除、メンバー管理、設定変更、棚卸しなどの管理業務を、効率的かつ安全に実施できるようにする。本プロジェクトでは、Google スプレッドシートと CSV を入出力とした Google Apps Script(GAS)ツールを開発し、一括操作による運用工数の削減と作業ミスの防止を図る。

## 2. システム概要

Google スプレッドシートを管理画面とし、GAS から Admin SDK(Directory API)および Groups Settings API を呼び出して Google グループを一括操作するツール。操作指示は「差分適用」方式(指示シートに記載した操作行のみを実行)とし、CSV による指示データの取込と結果データの出力に対応する。

## 3. 前提条件(確定事項)

| ID | 項目 | 内容 |
|----|------|------|
| P-1 | 実行者 | Google Workspace 特権管理者 |
| P-2 | 管理画面 | Google スプレッドシート(CSV の取込・出力に対応) |
| P-3 | 反映方式 | 差分適用(指示した操作のみ実行。シートを正とする完全同期は行わない) |
| P-4 | 実行方法 | 手動実行(カスタムメニュー)および定期自動実行(時間トリガー)の両方 |
| P-5 | 規模 | グループ数 1,000 超(メンバー延べ数は確認事項 C-1) |
| P-6 | 利用 API | Admin SDK Directory API、Groups Settings API(GAS 拡張サービス) |

## 4. スコープ

### 4.1 対象(In Scope)

- グループの作成・削除
- メンバーの追加・削除(ロール指定を含む)
- グループ設定の変更
- グループ・メンバー一覧の取得(棚卸し)とシート/CSV への出力
- CSV による指示データの取込・結果データの出力
- 実行ログの記録・エラーレポート

### 4.2 対象外(Out of Scope)

- ユーザーアカウント自体の作成・削除・変更
- 組織部門(OU)、共有ドライブなどグループ以外のリソース管理
- グループのメールアーカイブ(過去スレッド)の移行・管理
- 動的グループの条件管理(扱いは確認事項 C-7 参照)

## 5. 機能要件

| ID | 機能 | 要件内容 | 優先度 |
|----|------|----------|--------|
| FR-01 | 指示データ取込 | 指示シートから操作行(操作種別・対象・パラメータ)を読み込む。CSV ファイルからの取込にも対応する。 | 必須 |
| FR-02 | グループ作成 | メールアドレス・グループ名・説明を指定して作成する。 | 必須 |
| FR-03 | グループ削除 | 指定グループを削除する。誤削除防止のため確認用フラグ列を必須とする。 | 必須 |
| FR-04 | メンバー追加 | ロール(OWNER/MANAGER/MEMBER)を指定して追加する。追加済みの場合はスキップし結果に記録する。 | 必須 |
| FR-05 | メンバー削除 | 指定メンバーをグループから削除する。 | 必須 |
| FR-06 | グループ設定変更 | 投稿権限・参加権限・外部メンバー許可・閲覧権限などを変更する(対象項目の確定は確認事項 C-3)。 | 必須 |
| FR-07 | 棚卸し | 全グループの一覧と各グループのメンバー一覧をシートに出力する。 | 必須 |
| FR-08 | CSV 出力 | 棚卸し結果・実行結果を CSV としてドライブに出力する。 | 必須 |
| FR-09 | 事前バリデーション | 実行前にメール形式・必須項目・対象の存在有無・重複を検証し、不正な行は実行せず理由を記録する。 | 必須 |
| FR-10 | ドライラン | 変更系 API を呼び出さず、検証のみを行うモードを提供する。 | 必須 |
| FR-11 | 実行結果記録 | 行単位で成功/失敗/スキップの別、エラー内容、実行日時を指示シートに書き戻す。 | 必須 |
| FR-12 | 実行ログ | 実行履歴(実行者・日時・処理件数・結果概要)をログシートに記録する。 | 必須 |
| FR-13 | 手動実行 | スプレッドシートのカスタムメニューから各機能を実行できる。 | 必須 |
| FR-14 | 定期実行 | 時間主導型トリガーで定期実行できる(対象とする処理は確認事項 C-2)。 | 必須 |
| FR-15 | 完了・エラー通知 | 処理完了時およびエラー発生時にメール通知する(要否・宛先は確認事項 C-5)。 | 任意 |

## 6. 非機能要件

| ID | 区分 | 要件内容 |
|----|------|----------|
| NFR-01 | 性能・規模 | グループ 1,000 超・メンバー数万件規模を処理できること。一覧取得は API のページネーションを用いる。 |
| NFR-02 | 実行時間 | GAS の実行時間上限(1 実行あたり約 6 分)を考慮し、処理を分割して継続トリガーで自動再開し、全件を完了できること。 |
| NFR-03 | API 制限 | Directory API 等のレート制限超過時は、指数バックオフによるリトライを行うこと。 |
| NFR-04 | 冪等性 | 同一の指示を再実行しても安全であること(追加済み・削除済みはスキップ扱いとする)。 |
| NFR-05 | 安全性 | 削除などの破壊的操作は、確認フラグとドライランにより誤操作を防止すること。 |
| NFR-06 | 監査性 | 実行ログを一定期間(目安 1 年・要確認)参照できること。 |
| NFR-07 | セキュリティ | スプレッドシート・スクリプトへのアクセスを管理者に限定し、OAuth スコープは必要最小限とすること。 |
| NFR-08 | 保守性 | シート名・列構成・通知先などの設定値を、設定シートまたは定数として一元管理すること。 |

## 7. 制約事項

- GAS の実行時間は 1 実行あたり約 6 分が上限のため、大量処理は分割実行で対応する。
- Admin SDK・Groups Settings API の拡張サービス有効化と、管理コンソール側での API アクセス許可が必要。
- Directory API には日次・分次のクォータがあり、大量処理時は処理速度がクォータに律速される。
- スプレッドシートには 1 ファイルあたり 1,000 万セルの上限があるため、大規模な棚卸し結果は分割または CSV 出力で対応する。

## 8. 確認事項(オープン課題)

| ID | 確認事項 | 背景 |
|----|----------|------|
| C-1 | メンバー延べ数の規模感 | 処理の分割数・実行時間の見積りに必要。 |
| C-2 | 定期実行の用途 | 棚卸しの定期出力のみか、指示行の自動適用も含むか。自動適用は誤操作リスクがあり運用ルールが必要。 |
| C-3 | 変更対象とするグループ設定項目の一覧 | Groups Settings は項目が多数あるため、運用で使う項目に絞って設計したい。 |
| C-4 | 削除系操作の承認フロー | 確認フラグのみで足りるか、承認者列などを設けるか。 |
| C-5 | 通知の要否・宛先 | 完了・エラー時のメール通知の要否。 |
| C-6 | CSV の運用方法 | ドライブの特定フォルダに配置した CSV を取り込むか、手動でシートに貼り付けるか。 |
| C-7 | 動的グループ・セキュリティグループの有無 | 存在する場合、操作可否と対象範囲の定義が必要。 |
| C-8 | 利用者の拡大予定 | 将来、特権管理者以外(委任管理者など)が利用する可能性の有無。 |

## 9. 成果物

- GAS ソースコード一式
- 管理用スプレッドシート(指示・棚卸し・ログ・設定の各シートを含むテンプレート)
- 操作マニュアル(運用手順・エラー対応)
- テスト仕様書およびテスト結果報告

## 10. 開発の進め方

1. **要件定義:** 本書の確認事項を解消し、v1.0 として確定する。
2. **設計:** シートレイアウト、操作種別と列定義、分割実行方式、利用 API の仕様を確定する。
3. **コーディング:** 共通基盤(ログ・リトライ・分割実行)を先行実装し、その後に各機能を実装する。
4. **テスト:** テスト用グループ(または検証用ドメイン)で検証する。本番グループに対する破壊的操作のテストは行わない。
