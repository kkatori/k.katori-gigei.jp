# テスト仕様書：ユーザー管理機能開発

## 1. テスト計画・環境

### 1.1 テストの目的
本テストでは、東邦ホールディングス株式会社における「ユーザー管理機能」が要件定義書およびプログラム設計書に基づき、正確かつ安全に動作することを検証する。特に、Google Workspace (GWS) および rakumo API との整合性、冪等性の保証、大量データ処理時の性能、およびエラー発生時の整合性確保を重点的に確認する。

### 1.2 テスト対象範囲
- **フロントエンド**: AppSheet 操作画面（Action/Automation）
- **バックエンド**: Cloud Run API / バッチ処理（Python 3.12 / FastAPI）
- **データベース**: Cloud SQL (MySQL 8.0)
- **外部システム連携**: Google Workspace Admin SDK, rakumo Profile API, rakumo Google同期 API
- **インフラ**: Cloud Storage, Cloud Scheduler, Cloud Logging / BigQuery

### 1.3 テスト環境
- **検証用Google Cloudプロジェクト**: `toho-hd-user-mgmt-stg`
- **検証用Google Workspaceドメイン**: 検証用サブドメイン
- **検証用rakumoテナント**: 検証用サンドボックス
- **テストデータ**: 10,000ユーザー、10,000グループのダミーデータ（兼務10組織パターンを含む）

## 7. 判定基準
- **合格**: 全てのテスト項目が「期待値」通りに動作し、予期せぬシステムエラー（500系）が発生しないこと。
- **要再試行**: ネットワーク一時障害等でリトライが走り、最終的に成功した場合は「合格」とするが、リトライ回数が異常に多い場合はボトルネック調査を行う。
- **不合格**: データの不整合、重大なセキュリティ欠陥、または期待値に達しない性能遅延が発生した場合。
