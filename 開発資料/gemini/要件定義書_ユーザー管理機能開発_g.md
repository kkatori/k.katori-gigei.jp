# 要件定義書：ユーザー管理機能の開発

## 1. はじめに

### 1.1 背景
現在、社員情報のマスターデータとして「StarOffice」を利用しており、そこからCSVを出力して各業務システムへ連携を行っています。しかし、Google Workspaceとrakumoの導入に伴いStarOfficeが廃止されることとなり、「社員情報のマスタ」および「各システムへのCSV連携」の機能が失われることになります。

また、Googleドライブのアクセス権限管理のためにGoogleグループを作成していますが、rakumoコンタクトの組織階層表示にこれを利用すると実態と乖離が生じます。用途に応じたグループ管理が必要となり、運用工数の増大が懸念されています。

### 1.2 目的
StarOffice廃止後の代替となるユーザーマスタ機能をGoogle Cloud上に構築し、以下の実現を目指します。
- **ユーザーマスタ機能の構築**: Google Cloud上にデータベース（Cloud SQL）を構築し、社員情報を一元管理する（シングルソースオブトゥルース）。
- **運用効率化**: Google Workspaceのアカウントおよびグループの一括登録・更新・削除を自動化・効率化する。
- **rakumo連携**: rakumoプロファイルAPIを活用し、コンタクト情報の更新・同期を自動化する。
- **他システム連携**: ユーザーマスタから他システム連携用のCSVを作成・ダウンロード可能にする。
- **定期実行・予約実行**: 各種連携処理の定期実行や予約実行を可能にする。

## 2. システム概要

### 2.1 システム構成
本システムはGoogle Cloud Platform (GCP) を基盤とし、フロントエンドにAppSheetを採用します。

- **フロントエンド**: AppSheet (ユーザー/グループ一覧表示、入力画面、ログ表示)
- **バックエンド**: Google Cloud (Cloud Run, Cloud Tasks, Cloud Scheduler)
- **データベース**: Cloud SQL, Firestore (想定)
- **ストレージ**: Cloud Storage (CSVファイルのアップロード/ダウンロード用)
- **連携先**:
  - Google Workspace (Admin SDK Directory API)
  - rakumo (rakumo プロファイルAPI)
  - ファイルサーバー/ローカルシステム (CSV連携)

### 2.2 データフロー
1. AppSheetまたはCSV経由でユーザー/グループ情報を入力。
2. Cloud StorageへCSVをアップロード。
3. Cloud Run/App Engine上のアプリケーションがCSVを読み込み、データベース (Cloud SQL) を更新。
4. Google WorkspaceおよびrakumoへAPI経由で情報を反映。
5. 他システム連携用CSVを生成し、署名付きURL経由でダウンロード可能にする。

## 3. 機能要件

### 3.1 Web画面機能 (AppSheet想定)
ユーザーおよび管理者が操作するインターフェースを提供します。
- **一覧表示**: ユーザーおよびグループの一覧を表示。
- **情報登録・編集**: ユーザー・グループ情報の新規登録、編集画面。
- **CSVアップロード**: ユーザー用、グループ用のCSVファイルをアップロードする機能。
- **CSVダウンロード**: 登録済み情報のCSVダウンロード、他システム連携用CSVのダウンロード。
- **同期・更新実行**: Google同期、rakumoプロファイルアップデートの手動実行ボタン。
- **ログ表示**: 各機能の実行ログ（成功・失敗、日時、実行者）を表示。

### 3.2 Googleアカウント管理機能
- **CSV一括処理**: アップロードされたCSVに基づき、アカウントの追加・変更・削除を一括で実行。
- **差分同期による冪等性の確保**: **実行時に現在の状態（Workspace上のデータ）とDB入力を比較し、必要な変更のみを適用する「差分同期方式」を採用する。これにより、再試行時の冪等性を担保する。**
- **データベース登録**: Google Workspaceのユーザー情報をGCP上のデータベースに登録・同期。
- **アカウント一覧出力**: Google Workspaceのアカウント情報をCSVとして出力。

### 3.3 Googleグループ管理機能
- **CSV一括処理**: グループおよびグループメンバーの追加・変更・削除を一括で実行。
- **差分同期の適用**: **ユーザー管理と同様に差分のみを更新するロジックとし、APIリクエストの削減と冪等性を実現する。**
- **用途別グループ管理**:
    - Googleドライブ権限管理用グループ
    - メーリングリスト用グループ
    - rakumo表示用グループ (プレフィックスによる識別管理を想定)
- **データベース登録**: Google Workspaceのグループ情報をGCP上のデータベースに登録。

### 3.4 Google同期実行機能
- **同期実行**: Google Workspaceとrakumo間の同期処理を実行（API経由または自動連携）。
- **スケジュール実行**: 指定された時間または定期的なタイミングでの同期実行。

### 3.5 rakumoプロファイル更新機能 (rakumo API連携)
rakumo プロファイルAPIを利用して、コンタクト情報を更新します。
- **認証処理**:
  - APIキーと秘密鍵を用いたHMAC-SHA1署名による認証（Authorizationヘッダ: `RWS [APIKEY]:[SIGNATURE]`）を実装。
- **CSV生成**: rakumo指定フォーマットのCSVを生成。
- **データ更新フロー**:
  1. `CSV IMPORT LOCK` APIにより、他の更新処理をロック。
  2. `CSV CREAT URL` APIにより、アップロード用URLを取得。
  3. `CSV UPLOAD` APIを用いてCSVファイルをアップロード。
  4. 処理完了後、`CSV IMPORT UNLOCK` APIによりロックを解除。
- **運用制御**:
  - 大きなCSVファイルの転送は数時間に1回程度に抑えるよう、システム側で流量を制御する。
  - レート制限およびエラーハンドリング（400/401/50x系）の実装。

### 3.6 他システム連携用CSVダウンロード機能
- **CSV生成**: データベースの情報から、他システム連携に必要なフォーマットでCSVを生成。
- **ダウンロード**: **生成されたCSVをブラウザ経由でダウンロード（署名付きURL）し、ファイルサーバー等へ保存可能とする。自動保存については、ネットワーク環境およびセキュリティ要件に応じて将来的に検討する。**

## 4. 非機能要件

### 4.1 可用性
- **稼働時間**: 24時間365日（Google Cloudおよび連携サービスのメンテナンス時を除く）。

### 4.2 信頼性・保全性
- **データ整合性**: DB更新時のトランザクション管理を行い、エラー時はロールバックする。
- **再実行**: rakumo API等のエラー時は、指数関数的リトライ（Exponential Backoff）およびCloud Tasksによる再試行管理を行う。

### 4.3 セキュリティ
- **認証・認可**: Google Cloudの認証基盤を利用し、アクセス権限を管理。
- **APIキー管理**: rakumo APIキー等の機密情報は Secret Manager 等で安全に管理。
- **通信暗号化**: 全ての通信はHTTPSにより暗号化。

### 4.4 運用・保守
- **監視**: Cloud Loggingにより24時間365日のログ監視を行う。
- **アラート**: 重大なAPI連携失敗時に管理者へ通知。
