## 2. 開発アプローチ

### 統合先行開発を推奨

Tool 3（統合ツール）を先に開発し、Tool 1・2 をサブセットとして派生させる。

**選定理由**:

| 観点 | 逐次開発（A案） | 統合先行（B案）★推奨 |
|------|----------------|---------------------|
| 総期間 | 12-14週 | **10-12週** |
| コード重複 | 3つの独立コードベース | **1コードベースから3成果物** |
| 保守コスト | 高い | **低い** |
| 早期リリース | Tool 1が5月末に可能 | 全ツール7月同時 |
| 設計品質 | 後からの統合が困難 | **最初から統合設計** |

Tool 3 = Tool 1 + Tool 2 の上位集合であることが仕様上明確であり、機能フラグ（`TOOL_MODE`）による出し分けで3製品を1つのコードベースから生成できる。

## 4. フェーズ詳細

### Phase 1: 設計・共通基盤（W1-W3, 5/12-5/30）

#### W1（5/12-5/16）: 要件定義・アーキテクチャ設計

| 作業項目 | 成果物 |
|----------|--------|
| シート構成設計 | ユーザー一覧、グループ一覧、設定、ログの各シート定義 |
| CSVカラムマッピング | rakumo 3種CSV（ユーザー詳細/ライセンス/部門）の列定義 |
| モジュール構成設計 | 依存関係図、インタフェース定義 |
| 機能フラグ設計 | `TOOL_MODE = 'rakumo' \| 'gws' \| 'integrated'` の仕様 |
| ドキュメント | `docs/要件定義書.md`, `docs/アーキテクチャ設計書.md` |

#### W2-W3（5/19-5/30）: 共通モジュール開発

| モジュール | ファイル | 内容 |
|-----------|---------|------|
| 認証 | `shared/src/auth.js` | rakumo HMAC-SHA1署名生成 |
| 設定管理 | `shared/src/config.js` | PropertiesServiceベースの設定読み書き |
| CSV処理 | `shared/src/csv.js` | CSV生成・パース・バリデーション |
| シート操作 | `shared/src/sheetUtils.js` | データ読み書き、ステータス表示、メニュー構築 |
| HTTP通信 | `shared/src/apiClient.js` | リトライ、レート制限、エラーハンドリング |
| ログ | `shared/src/logger.js` | ログシートへの実行記録出力 |

---

### Phase 2: Tool 3 コア実装（W3-W7, 5/26-7/03）

#### W3-W4（5/26-6/06）: rakumo API連携

| モジュール | 主要機能 |
|-----------|---------|
| `rakumoProfile.js` | CSV Download（現在のユーザー情報取得）、CSV Upload（Lock→URL取得→Upload→Unlock） |
| `rakumoSync.js` | Google同期ジョブ実行（CREATE）、状態監視（GETCURRENT / GETLASTRESULT） |

**技術的注意点**:
- Upload フローは Lock → URL取得 → Upload → Unlock の順序を厳守。`finally` で Unlock を保証
- 同期完了のポーリング時、GAS 6分制限を考慮しトリガー分割を検討

#### W4-W5（6/02-6/13）: GWS Admin SDK連携

| モジュール | 主要機能 |
|-----------|---------|
| `gwsUsers.js` | ユーザー作成・更新・削除・停止、一覧取得（ライセンス情報含） |
| `gwsGroups.js` | グループCRUD、メンバー管理、アクセス権更新、エイリアス、アドレスリスト表示切替 |

**技術的注意点**:
- Admin SDK は GAS の Advanced Service として利用（`appsscript.json` に設定追加）
- ページネーション処理（`nextPageToken`）
- レート制限対応（Admin SDK クォータ: 2400 req/min）

#### W5-W6（6/09-6/20）: スプレッドシートUI + CSV生成

| モジュール | 主要機能 |
|-----------|---------|
| `ui.js` | カスタムメニュー（モード別出し分け）、プログレス表示、行ステータス（成功/失敗/スキップ） |
| `csvGenerator.js` | ユーザー詳細CSV、ライセンス割り当てCSV、部門設定CSV、rakumoフォーマット変換 |

#### W6-W7（6/16-7/03）: 統合フロー

| モジュール | 主要機能 |
|-----------|---------|
| `workflows.js` | GWS登録→Google同期→rakumo CSVの一連フロー、一括登録/更新/削除、エクスポート |

**技術的注意点**:
- GAS 6分制限対応: バッチ分割 + トリガー連鎖、PropertiesService に処理位置を保存
- 組織階層マッピング: GWS の組織単位 (OU) ↔ rakumo の部門・職制
- 冪等性: 途中失敗時の再実行を安全に行える設計

## 6. リスクと対策

| リスク | 影響度 | 発生確率 | 対策 |
|--------|--------|----------|------|
| GAS 6分実行制限 | 高 | 高 | バッチ分割 + トリガー連鎖、PropertiesService に処理位置保存 |
| rakumo CSVフォーマット不明瞭 | 高 | 中 | Phase 1 で sample CSV 詳細分析、不明点は rakumo サポートに早期確認 |
| Admin SDK クォータ制限 | 中 | 中 | `Utilities.sleep()` によるレート制限、指数バックオフ |
| テスト環境未確保 | 高 | 中 | **開発着手前に確保（ブロッカー）** |
| rakumo API 認証タイミングずれ | 中 | 低 | UTC 時刻の正確な生成、±15分制限のテスト検証 |
