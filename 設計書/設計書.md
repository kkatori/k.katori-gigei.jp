## 3. 処理フロー

### 3-1. メインフロー（rakumoSync.bat）

```
[START]
	│
	▼
[1] user.bat
	AD から有効ユーザー一覧を src/users.csv へエクスポート
	│
	▼
[2] group.bat
	AD から rakumo OU のグループ一覧を src/groups.csv へエクスポート
	│
	▼
[3] convertdata_JTC.ps1
	users.csv + groups.csv を Rakumo インポート形式（ad_users.csv）に変換
	│
	▼
[4] main.exe
	ad_users.csv を読み込み、Rakumo API へ送信・同期実行
	│
	▼
[END] ※エラー時は sendMail.ps1 でエラー通知メール送信
```

### 3-2. 各ステップの詳細

#### STEP 1: user.bat — ADユーザー抽出

```
csvde -u -f .\src\users.csv \
	-r "(&(objectClass=user)(!userAccountControl:1.2.840.113556.1.4.803:=2))" \
	-d "OU=rakumo,DC=ad,DC=gigei,DC=jp"
```

| パラメータ | 説明 |
|---|---|
| `-u` | Unicode 形式で出力 |
| `-f .\src\users.csv` | 出力先ファイル |
| `-r (フィルタ)` | objectClass=user かつ userAccountControl の「無効」ビットが立っていないもの（有効アカウントのみ） |
| `-d "OU=rakumo,..."` | 検索基点 DN |

#### STEP 2: group.bat — ADグループ抽出

```
csvde -u -f .\src\groups.csv \
	-r "(&(objectCategory=group))" \
	-d "OU=rakumo,DC=ad,DC=gigei,DC=jp"
```

| パラメータ | 説明 |
|---|---|
| `-r (フィルタ)` | objectCategory=group（グループオブジェクト全件） |

#### STEP 3: convertdata_JTC.ps1 — データ変換

**入力ファイル:**
- `src/users.csv` — AD ユーザー情報（csvde 出力、Unicode）
- `src/groups.csv` — AD グループ情報
- `src/rakumo2ad.config` — AD 属性 → Rakumo プロパティ マッピング
- `src/sort_properties.config` — ソート対象プロパティ
- `src/JTC_properties.config` — 職務コード設定
- `config.tml` — Rakumo同期ツールメイン設定

**出力ファイル:**
- `src/ad_users.csv` — Rakumo インポート用 CSV（UTF-8）

**主要処理:**

| # | 処理 | 詳細 |
|---|---|---|
| 1 | バリデーション | csvcheck.ps1 を呼び出して入力 CSV の整合性チェック |
| 2 | グループ展開 | ユーザーが所属する各グループに対して 1 レコードずつ展開 |
| 3 | メールアドレス付与 | グループ DN から JTC_groupmail.config を参照してメールアドレスをマッピング |
| 4 | 職務コード付加 | グループメールの条件に基づき JTC_properties.config から職務コードを追加 |
| 5 | プロパティ変換 | rakumo2ad.config の定義に従い AD 属性名を Rakumo プロパティ名に変換 |
| 6 | 検証 | ユーザー ID・部門メールアドレスの形式検証 |
| 7 | 文字列処理 | メールアドレスへのサフィックス追加（設定ファイルに基づく） |
| 8 | 出力 | UTF-8 形式で ad_users.csv に書き出し |

#### STEP 4: main.exe — Rakumo API 同期

変換済み `ad_users.csv` を読み込み、Rakumo API へ送信してユーザー情報を同期する。

## 4. Rakumo API 認証仕様（rakumoAPI.ps1）

| 項目 | 仕様 |
|---|---|
| 認証方式 | HMAC-SHA1 署名（RWS 認証） |
| 認証ヘッダ形式 | `RWS {APIキー}:{HMAC-SHA1署名}` |
| 署名入力 | HTTPメソッド + Content-Type + 日時（RFC1123）を結合した文字列 |
| 日時ヘッダ | UTC 時刻（RFC1123 形式） |
| 通信プロトコル | HTTPS |
| 文字コード | UTF-8 |

**主要関数:**

| 関数名 | 役割 |
|---|---|
| `Get-RakumoDateHeader` | RFC1123 形式の UTC 時刻ヘッダを生成 |
| `Get-Signature` | メソッド・Content-Type・日時から HMAC-SHA1 署名を計算 |
| `Get-RequestHeader` | API リクエストヘッダ辞書を構築 |
| `Call-RakumoAPI` | HTTP 通信を実行し UTF-8 レスポンスを返却 |
| `Get-ContactProfile` | 連絡先プロファイルを CSV 形式でダウンロード |

## 6. エラーハンドリング

| シナリオ | 処理 |
|---|---|
| AD からのデータ取得失敗（csvde エラー） | バッチが異常終了。sendMail.ps1 でエラー通知メール送信（エラーコード3） |
| CSV バリデーションエラー | csvcheck.ps1 が異常終了。処理を中断 |
| メールアドレス未設定ユーザー | エラーコード1 で通知メール送信 |
| Rakumo API 通信エラー | エラーコード2 で通知メール送信 |
| 通知先メール送信失敗 | ログ記録（詳細は sendMail.ps1 実装依存） |
